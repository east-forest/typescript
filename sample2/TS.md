# undefiend と null の違い

- 共に値がないに相当する表現
- undefiend
  - 値が代入されていないため、値がない
  - 初期値で入れられるのは「undefiend」。そのため、自然発生する。
  - 変数
- null
  - 代入すべき値がないため、値がない
  - 実装者が意図しないと発生しない。
  - リテラル 変数名として作成できない
- 両者の使い分け
  - 基本的には、「undefiend」を利用するのが良い。
    - null は、開発者側で定義する必要があるので、記述量が多くなるデメリットがある。
- unknown と any
  - unknown は、値は入れることができるが、利用するときには制限が出てくる。
  - any は、上記のような制限はない。
    - if(typeof unknown 型 === 何か値)みたいな使い方で、利用できるようになる。
